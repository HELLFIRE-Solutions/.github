# Contributing to hellfire-ai

## Verification mechanism

Public contributions to `hellfire-ai/*` repositories are part of how HELLFIRE AI Solutions verifies contractors before admitting them to the delivery pool. A merged, reviewable pull request that demonstrates a relevant implementation of the module you want to work on stands in for a résumé or diploma — evaluated for whether it demonstrates a real, working implementation relevant to the module/vector you want to be considered for.

This applies to public repos only. `internal-db` and any repo holding client data are private and out of scope for external contributions.

## How to contribute

1. Open an issue describing what you plan to implement before starting substantial work.
2. Fork the repo, implement against the module's README (dogfood step first, template step second).
3. Open a PR referencing the issue. Keep it scoped — one module, one concern.

## Code of conduct

Be direct, be respectful, no harassment. Violations result in removal from the contributor pool.
